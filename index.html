1. Tư duy về bài

Bài toán lập trình không phải học thuộc lời giải, mà là học cách:

  • Phân tích dữ liệu vào – ra

III. Phương pháp làm bài

Hai con trỏ (Two Pointers)

1. Khái niệm

Dùng hai con trỏ để duyệt mảng nhằm giải quyết bài toán nhanh hơn và giảm độ phức tạp.

2. Các dạng

Dạng 1: Hai đầu mảng (Opposite Pointers)

Cách hoạt động:

  • Đặt hai điểm ở hai đầu mảng đã sắp xếp từ bé đến lớn
  • Nếu kết quả > đáp án, giảm giá trị điểm cuối
  • Nếu kết quả < đáp án, tăng giá trị điểm đầu
  • Nếu tìm thấy đáp án, dừng
  • Vòng lặp kết thúc mà chưa tìm được => không tồn tại cặp nào

    • Độ phức tạp: O(n) vì hai con trỏ đi tổng tối đa n bước.

Ví dụ:

Cho mảng a có n phần tử, tìm cặp số trong mảng a có tổng bằng x.

n = 4
a = [1, 3, 5, 10]
x = 8

Đặt hai đầu mút là l = 0 và r = n-1
l=0 (a[l]=1), r=3 (a[r]=10) → 11 > 8 → giảm r
l=0 (a[l]=1), r=2 (a[r]=5)  → 6  < 8 → tăng l
l=1 (a[l]=3), r=2 (a[r]=5)  → 8  ✓

#include 
using namespace std;

bool twoSum(vector& a, int x) {
    int l = 0, r = a.size() - 1;
    while (l < r) {
        int s = a[l] + a[r];
        if (s == x) return true;
        if (s < x) l++;
        else r--;
    }
    return false;
}

def two_sum(arr, x):
    l, r = 0, len(arr) - 1
    while l < r:
        s = arr[l] + arr[r]
        if s == x:
            return True
        if s < x:
            l += 1
        else:
            r -= 1
    return False

Dạng 2: Cửa sổ trượt (Sliding Window)

Cách hoạt động:

  • Hai điểm ở đầu mảng, một điểm giữ đầu mảng con, một điểm mở rộng mảng con
  • Nếu thoả mãn điều kiện, tăng điểm bên phải mở rộng mảng con
  • Nếu không thoả mãn điều kiện, tăng điểm bên trái giữ đầu mảng con
  • Trong khi thoả mãn điều kiện, tạo một biến lưu kết quả (lớn nhất)

    • Độ phức tạp: O(2n) vì mỗi con trỏ chạy tối đa n bước,

Ví dụ

Cho mảng a có n phần tử, tìm độ dài mảng con liên tiếp dài nhất có tổng ≤ s.

n = 6
a = [1, 3, 10, 5, 6, 1]
s = 12
ans = -2e9 (số vô cùng nhỏ để so sánh max)

Đặt hai đầu mút là l = 0 và r = 1, với mỗi vòng lặp nếu thỏa mãn tổng ≤ s thì ans = max(ans, r - l + l)
*r - l + 1 là độ dài mảng con, +1 vì tính cả l.

l=0, r=0 → cộng từ a[0] đến a[0] → 1  ≤ 12 → tăng r, ans = max(-2e9, 1) = 1
l=0, r=1 → cộng từ a[0] đến a[1] → 4  ≤ 12 → tăng r, ans = max(1, 2) = 2
l=0, r=2 → cộng từ a[0] đến a[2] → 14 > 12 → tăng l
l=1, r=2 → cộng từ a[1] đến a[2] → 13 > 12 → tăng l
l=2, r=2 → 10 ≤ 12 → tăng r, ans = max(2, 1) = 2
l=2, r=3 → 15 > 12 → tăng l
l=3, r=3 → 5  ≤ 12 → tăng r, ans = max(2, 1) = 2
l=3, r=4 → 11 ≤ 12 → tăng r, ans = max(2, 2) = 2
l=3, r=5 → 12 ≤ 12 (kết thúc vì đã chạy hết mảng a), ans = max(2, 3) = 3
Vậy độ dài mảng con dài nhất là: 3

#include 
using namespace std;

int smallarray(int n, vector& a, int s) {
    int l = 0, r = 0, t = 0, ans = -2e9;
    while (r < n) {
        int t += a[r];
        if (t <= s) {
          ans = max(ans, r - l + 1);
          r++;
        }
        else {
          t -= a[l]; #t trừ đi a[l] vì đầu mảng con (l) tăng lên nên sẽ bỏ đi phần tử a[l] cũ
          l++;
        }
    }
    return ans;
}

def smallarray(n, arr, s):
    l, r, t, ans = 0, 0, 0, -float(inf)
    while r < n:
        t += arr[r]
        if t <= s:
            ans = max(ans, r - l + 1)
            r += 1
        else:
            t -= arr[l]
            l += 1
    return ans

3. Mẹo & từ khóa

Từ khóa: mảng đã sắp xếp, tìm cặp, đoạn con liên tiếp, tối ưu O(n).

Dynamic Programming (Quy hoạch động)

1. Khái niệm

Quy hoạch động (Dynamic Programming) là phương pháp chia bài toán lớn thành các bài toán con nhỏ hơn, lưu lại kết quả của các bài toán con để tránh tính toán lặp lại.

2. Ví dụ

Đếm số cách leo lên cầu thang có n bậc, mỗi lần đi 1 hoặc 2 bậc.

Ý tưởng làm bài

Gọi dp[i] là số cách đi tới bậc thứ i

- Để tới bậc i:
  + Có thể đi từ bậc i-1 (đi 1 bước)
  + Hoặc từ bậc i-2 (đi 2 bước)

=> dp[i] = dp[i-1] + dp[i-2]

Điều kiện ban đầu:
- dp[0] = 1 (đứng yên)
- dp[1] = 1 (chỉ có 1 cách)

Tính dp tăng dần từ nhỏ tới lớn

Độ phức tạp:
- Thời gian: O(n)
- Bộ nhớ: O(n) (có thể tối ưu xuống O(1))

3. Các dạng thường gặp

  • DP 1 chiều (Fibonacci, cầu thang)
  • DP 2 chiều (xâu con chung, balo)
  • DP trên đoạn
  • DP trên cây / đồ thị

#include 
using namespace std;

int main() {
    int n; cin >> n;
    vector dp(n+1);
    dp[0] = 1;
    if (n >= 1) dp[1] = 1;

    for (int i = 2; i <= n; i++)
        dp[i] = dp[i-1] + dp[i-2];

    cout << dp[n];
}

n = int(input())

dp = [0] * (n + 1)
dp[0] = 1
if n >= 1:
    dp[1] = 1

for i in range(2, n + 1):
    dp[i] = dp[i - 1] + dp[i - 2]

print(dp[n])

5. Mẹo & từ khóa

Từ khóa nhận biết: bài toán tối ưu, đếm số cách, có trạng thái phụ thuộc vào trạng thái trước, subtask nhỏ ăn được bằng brute force.
